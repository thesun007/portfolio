ImSimViewer

상부 요청을 바탕으로 Physical AI 시뮬레이션 결과를 Unreal 월드에서 확인하기 위한 Viewer 구성을 설계하고 구현한 프로젝트. 핵심은 특정 데모 하나가 아니라, 외부 시뮬레이터 데이터가 들어왔을 때 맵, Entity, View, 환경을 설정 기반으로 확장할 수 있는 시각화 플랫폼 구조를 만드는 것

역할 단독 개발
엔진 Unreal C++
연동 ROS2 Humble · ZMQ · glTF · XML
성격사내 자체 프로젝트
목표시뮬레이션 시각화 플랫폼
범위구조 설계 · 구현 · 로컬 검증
작업 관점 요청사항을 Viewer 구조로 구체화

Physical AI 시뮬레이션 결과를 Unreal에서 확인하기 위한 요청을 바탕으로, 맵·Entity·View·환경 제어가 설정과 데이터 흐름에 따라 동작하는 Viewer 구조로 정리

설정 기반 구성

맵, Entity, glTF, View 정보를 XML 설정으로 분리해 패키지 이후 확장 가능하도록 구성

통신 전제 대응

ROS2/ZMQ 실시간 테스트가 제한된 상황에서 로컬 UI와 기록 데이터 재생으로 처리 흐름 검증

시각화 품질 확보

렌더링 옵션, 플러그인, 샘플 파티클 편집을 조합해 차량·기상·해상·수중 표현을 구성

작업 방향

01 설정 로드

XML에서 map, view, entity, glTF 정보를 읽고 런타임 스폰 준비

02 ROS2Bridge

rclUE를 Windows Unreal 환경에 맞게 재구성하고 rcl C API 기반 송수신 구조 준비

03 제어 서비스

월드, 시뮬레이션, Entity, 환경 제어 매니저가 메시지를 처리

04 ZMQ 차량 처리

차량 텔레메트리로 이동, 물리 상태, 주행 효과를 반영

05 환경 시각화

렌더링 옵션, 날씨, 차량 효과, 해상/수중 표현을 Unreal 월드에 반영

06 로컬 검증

실시간 연동 전 단계에서 UI와 기록 데이터 재생으로 동작 흐름 확인

담당 상세

구현 상세

ROS2Bridge, 제어 서비스, 차량 효과, 환경 제어, 해상/수중 검증으로 나누어 정리

ROS2Bridge / 제어 서비스

ROS2Bridge 플러그인을 기반으로 월드, 시뮬레이션, Entity, 환경 상태를 처리하는 서비스와 매니저 클래스를 구성

ROS2Bridge

rclUE 기반 구조를 Windows Unreal 환경에 맞게 재구성. C++ symbol mangling 문제로 rclcpp 대신 rcl C API를 직접 사용하고, 반복 사용성을 보완하기 위해 매크로 구성을 준비

Node 구조

액터에 부착하는 컴포넌트 단위 노드와 게임 인스턴스 단위 글로벌 노드를 제공. rclUE에 없던 글로벌 ROS Node 기능을 보완

메시지 호환

표준 ROS2 메시지와 커스텀 메시지 패키지(simul_msgs)를 사용할 수 있도록 구성하고, 메시지 정의와 노드 사용 절차를 매뉴얼로 정리

제어 서비스

월드/시뮬레이션 제어, Entity 컨트롤, 날씨/환경 컨트롤 서비스를 구성하고 각 메시지를 매니저 클래스에서 처리

설정 로드

ig config.xml, entity registry.xml로 맵, View, Entity, glTF 정보와 기타 옵션을 정의해 패키지 이후에도 시각화 구성을 교체할 수 있도록 구성

런타임 스폰

glTFRuntime 기반으로 Entity 모델을 로드하고, Entity/ViewActor 클래스를 통해 월드에 배치. glTF 모델 제작 가이드 문서 제공

ZMQ / 차량 효과

ZMQ Telemetry

차량 텔레메트리 포맷을 기준으로 이동 샘플 데이터를 파일 재생 방식으로 재현하고, 차량 이동 처리 경로를 검증

Vehicle VFX

주행, 슬립, 드리프트 활성 여부와 강도/방향을 판단해 Tire Trail, 지형/눈/비 조건별 차량 효과를 적용

환경 제어 / 렌더링

환경 제어

로컬 UI로 시간, 기상, 풍속, 안개, 재질 젖음/적설/먼지 상태를 조정하고 월드 환경에 반영

렌더링 옵션

Lumen, Nanite, VSM 기준 렌더링 품질 옵션을 조정해 시각화 플랫폼에서 필요한 품질과 성능 기준을 확인

해상 / 수중 환경 검증

Unreal Viewer에서 수면, 수중 카메라, 깊이별 분위기, 잠수정 조작, 대형 선박 부력까지 다룰 수 있는지 확인

수면과 수중 표현 구성

UE Water Plugin과 Ultra Dynamic Sky 플러그인으로 수면, 수중 카메라, 기상/안개 기반 표현을 구성. Caustics와 God Ray는 플러그인 효과를 활용하고, 화면 일렁임은 별도 Post Process 로직으로 추가

깊이별 수중 분위기

카메라의 수면 대비 깊이를 기준으로 색 채널별 감쇠율 차등 원리를 적용해 깊이별 수중 분위기를 조정. 수면 위 이동, Pawn 전환, 종료 시 변경 값을 원래 상태로 복원

잠수정과 부력 검증

HoloOcean을 통합해 HoloOcean agent 기반 잠수정 조종과 센서 연동을 테스트. Water Plugin 부력 컴포넌트와 질량, 선/각속도 damping 값을 조절해 구축함/항공모함급 선박 모의 테스트 진행

확인한 한계

UE Water Plugin을 사용할 경우 underwater Post Process가 반투명 표현과 light scattering을 가리는 문제가 있어, 관련 보정은 추가 개발 과제로 정리
